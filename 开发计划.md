# 开发计划

## 旅游模块后续计划

### 1. 接入旅游订单的微信小程序支付闭环
- 状态：已完成
- 范围：
  - 后端新增旅游订单创建支付参数接口。
  - 后端新增旅游订单支付确认接口。
  - 微信支付回调支持识别旅游订单并更新旅游报名、旅游订单、排期支付人数。
  - 小程序“我的旅游报名”页接入 `wx.requestPayment`。
- 验收：
  - 本地 mock 支付可直接完成旅游报名缴费。
  - 真实微信支付模式下可生成 JSAPI 支付参数。
  - 支付成功后报名状态变为 `paid`，支付状态变为 `paid`。
- Review：
  - 已复用现有微信支付 JSAPI 下单、查单、通知验签和解密能力。
  - 微信支付通知通过旅游订单号前缀 `LYP` 分流，不影响原商品订单。
  - 支付成功处理具备幂等保护，避免回调和前端确认重复更新。

### 2. 增加成团/缴费订阅消息
- 状态：已完成
- 范围：
  - 小程序在报名或缴费入口前请求订阅消息授权。
  - 后端在成团通知缴费时发送站内消息，并预留订阅消息发送服务。
  - 支持通过环境变量配置旅游缴费订阅模板 ID。
- 验收：
  - 未配置模板 ID 时不影响原站内消息。
  - 配置模板 ID 后，成团通知缴费流程会尝试发送订阅消息。
- Review：
  - 小程序会在报名提交和缴费前请求缴费提醒订阅授权。
  - 后端通过 `WECHAT_TRAVEL_PAYMENT_SUBSCRIBE_TEMPLATE_ID` 控制是否发送。
  - 订阅消息发送失败不阻断后台成团通知主流程。

### 3. 后台报名列表增加全量导出和按排期导出
- 状态：已完成
- 范围：
  - 后端报名列表支持 `scheduleId` 过滤。
  - 后台报名列表增加排期筛选。
  - 后台导出支持当前筛选条件下全量导出，不再只导出当前页。
  - 线路排期处增加按排期跳转查看报名。
- 验收：
  - 可导出当前筛选条件下全部报名。
  - 可按排期查看并导出报名名单。
- Review：
  - 后端 `scheduleId` 仅在有效数字时参与过滤，普通列表不受影响。
  - 前端全量导出按每页 100 条分批拉取，符合后端分页限制。
  - 线路排期“看报名”通过 query 参数跳转，可直接复用报名列表。
