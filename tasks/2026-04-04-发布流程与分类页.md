# 发布流程与分类页

## 任务目标

基于微信小程序原生技术栈，补充首页发布菜单后的分类选择页与通用发布页，并删除旧的静态原型文件。

## 设计参考说明

- 首页点击右下角加号后，选择某个发布类型
- 先进入“选择分类”页面
- 选中详细分类后进入“发布页面”
- 发布页面整体结构统一，按不同类型和分类显示不同标题与字段文案
- 本次优先参考“树洞”截图实现视觉结构

## 页面模块拆解

- 首页发布菜单点击逻辑
- 分类选择页 `pages/category-select/category-select`
- 通用发布页 `pages/publish/publish`
- 发布类型与分类映射数据
- 删除旧静态原型文件

## 技术方案

- 使用统一配置映射发布类型、颜色、分类集合和发布页标题
- 分类选择页通过 query 传入 `type`
- 发布页通过 query 传入 `type` 与 `category`
- 发布页结构复用，按不同类型渲染分类标题和表单提示

## 实现方案

1. 删除旧静态原型文件
2. 在 mock 数据中新增发布类型与分类配置
3. 新增分类选择页
4. 新增通用发布页
5. 修改首页发布菜单点击跳转
6. 自检页面路径、参数传递和语法
