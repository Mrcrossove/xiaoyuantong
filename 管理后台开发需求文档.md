# 校园通管理后台开发需求文档

## 1. 文档目标

本文档用于统一校园通 Web 管理后台的产品范围、数据结构、权限模型、页面原型和前端工程组织方式。

后续后台开发以本文档为准，优先保证：

- 与当前小程序端业务一致
- 支持高校维度的数据隔离与管理
- 支持帖子、店铺、认证、消息、运营配置的完整管理
- 支持后续扩展到订单、钱包、提现和统计分析

---

## 2. 项目背景

当前小程序已具备以下业务模块：

- 首页帖子流
- 创业店铺页
- 商家分类页
- 店铺详情页
- 消息页
- 高校切换与全局联动
- 校园认证
- 我的页及其子页面

因此管理后台定位不是单纯的商城后台，而是：

`校园内容运营后台 + 商家管理后台 + 审核后台 + 高校管理后台`

---

## 3. 后台建设目标

### 3.1 第一阶段目标

完成后台基础运营能力，使小程序从静态演示进入可运营状态：

- 管理高校
- 管理用户
- 管理校园认证
- 管理帖子
- 管理店铺
- 管理商品
- 管理消息通知
- 配置 banner、推荐位、帮助信息
- 管理管理员与角色权限

### 3.2 第二阶段目标

补齐交易与数据能力：

- 订单系统
- 钱包与提现
- 退款处理
- 数据统计大盘
- 举报与风控
- 操作日志和审计

## 8. 后台权限角色设计

## 8.1 角色划分

建议至少 5 个角色：

### 1. 超级管理员

权限：

- 所有模块可见
- 所有数据可管理
- 管理后台账号、角色、菜单权限

### 2. 运营管理员

权限：

- 仪表盘
- 高校管理
- 用户管理查看
- 帖子管理
- 店铺管理
- 商品管理
- 消息通知
- 运营配置

不含：

- 管理员权限分配
- 核心系统设置

### 3. 审核员

权限：

- 校园认证审核
- 帖子审核
- 店铺审核
- 提现审核

不含：

- 运营配置
- 权限管理

### 4. 高校管理员

权限特点：

- 仅能查看和管理指定高校数据

权限范围：

- 本高校用户
- 本高校帖子
- 本高校店铺
- 本高校消息

### 5. 商家管理员

权限特点：

- 面向商家自己的后台

权限范围：

- 自己店铺信息
- 自己商品
- 自己订单
- 自己店铺消息

说明：

- 第一阶段后台如果先做平台管理端，可先不开发商家后台页面，但数据库与角色设计应预留。

## 8.2 权限控制维度

建议从 3 个维度做权限控制：

### 菜单权限

控制能否看到左侧菜单

### 按钮权限

控制能否执行：

- 审核
- 删除
- 上下架
- 推荐
- 导出

### 数据权限

控制能否查看：

- 全部高校
- 指定高校
- 仅自己创建的数据

## 8.3 权限表示例

### 超级管理员

- 全部菜单
- 全部按钮
- 全部数据

### 审核员

- 校园认证
- 帖子审核
- 店铺审核
- 提现审核

### 高校管理员

- 本校仪表盘
- 本校用户
- 本校帖子
- 本校店铺

## 12. 开发说明

后续后台开发应遵循以下原则：

- 先搭后台基础框架，再逐步接模块
- 优先做可运营能力，不先做重交易能力
- 所有列表页优先支持高校筛选
- 所有内容审核模块优先打通状态流转
- 所有运营内容优先可配置，不写死在前端
